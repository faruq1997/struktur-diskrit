üö∂ Apa itu Pencarian Beruntun?

Definisi

Algoritma Pencarian Beruntun (Linear Search) adalah cara yang paling sederhana dan paling dasar untuk mencari sebuah item dalam daftar (atau array).

Analoginya

Bayangkan Anda sedang mencari kaus kaki hitam tertentu di dalam laci yang berisi banyak kaus kaki yang tidak diurutkan.

üéØ Cara Kerja Pencarian Beruntun

Algoritma ini bekerja dengan cara:

  1. Mulai dari Awal: Dari item pertama dalam daftar
  2. Cek Satu Per Satu: Bandingkan item yang dicari dengan item saat ini
  3. Terus Maju: Jika tidak cocok, pindah ke item berikutnya
  4. Berhenti: Ketika item ditemukan atau daftar habis

üìä Contoh: Mencari Nama "Budi"

Daftar: [Andi, Siti, Dewi, Budi, Rina, Eko]

Langkah Item yang Dicek Perbandingan Hasil Aksi
1 Andi Andi == Budi? Salah Pindah ke berikutnya
2 Siti Siti == Budi? Salah Pindah ke berikutnya
3 Dewi Dewi == Budi? Salah Pindah ke berikutnya
4 Budi Budi == Budi? Benar! Pencarian Selesai

üí° Intinya:

Pencarian beruntun adalah algoritma pencarian paling dasar yang memeriksa setiap elemen secara berurutan hingga item ditemukan atau daftar habis.

üìè Array Berurut vs Tidak Berurut

Perbedaan Utama

Pencarian beruntun bekerja pada array berurut dan tidak berurut, tetapi ada optimasi khusus untuk array berurut.

Fitur Array Tidak Berurut Array Berurut
Susunan Data Elemen data acak, tidak ada pola Elemen data tersusun (misalnya, dari terkecil ke terbesar)
Cara Kerja Harus memeriksa semua elemen hingga item ditemukan atau daftar habis Dapat berhenti lebih awal jika item yang dicari lebih kecil dari elemen yang sedang diperiksa
Waktu Terburuk O(N) O(N) (sama-sama harus memeriksa N elemen jika item ada di akhir)
Optimasi Tidak ada optimasi yang bisa dilakukan Bisa dioptimasi (early exit) untuk kasus item tidak ada di tengah

üéØ Penghentian Awal (Early Exit)

Pada array berurut, kita bisa berhenti lebih awal. Misalnya mencari angka 15 di [5, 10, 20, 25]:

  • Cek 5 (15 > 5) ‚Üí Lanjut
  • Cek 10 (15 > 10) ‚Üí Lanjut
  • Cek 20 (15 < 20) ‚Üí STOP!

Karena 20 lebih besar dari 15, dan array sudah berurut, maka 15 pasti tidak ada di daftar ini.

üí° Intinya:

Array berurut memungkinkan penghentian awal yang menghemat waktu perbandingan pada kasus gagal pencarian.

üìù Contoh Kode Pencarian Beruntun

Berikut adalah implementasi algoritma pencarian beruntun dalam berbagai bahasa pemrograman:

üêç Python - Array Tidak Berurut

def linear_search_unsorted(arr, target): for i in range(len(arr)): if arr[i] == target: return i # Item ditemukan return -1 # Item tidak ditemukan # Contoh penggunaan names = ["Andi", "Siti", "Dewi", "Budi", "Rina"] result = linear_search_unsorted(names, "Budi") print(f"Budi ditemukan di indeks: {result}")

üêç Python - Array Berurut (Early Exit)

def linear_search_sorted(arr, target): for i in range(len(arr)): if arr[i] == target: return i # Item ditemukan elif arr[i] > target: return -1 # Early exit - item pasti tidak ada return -1 # Item tidak ditemukan # Contoh penggunaan numbers = [5, 10, 20, 25, 30] result = linear_search_sorted(numbers, 15) print(f"15 ditemukan di indeks: {result}")

‚òï Java - Array Tidak Berurut

public static int linearSearchUnsorted(int[] arr, int target) { for (int i = 0; i < arr.length; i++) { if (arr[i] == target) { return i; // Item ditemukan } } return -1; // Item tidak ditemukan } // Contoh penggunaan int[] numbers = {5, 2, 8, 1, 9}; int result = linearSearchUnsorted(numbers, 8); System.out.println("8 ditemukan di indeks: " + result);
üìä Analisis Kompleksitas

Pencarian beruntun memiliki kompleksitas waktu yang sederhana:

Kasus Kompleksitas Waktu Penjelasan
Terbaik O(1) Item ditemukan di posisi pertama
Rata-rata O(N) Item ditemukan di tengah array
Terburuk O(N) Item tidak ada atau di posisi terakhir

üí° Intinya:

Pencarian beruntun sangat cocok untuk array kecil atau array yang sering berubah. Untuk array besar yang sudah berurut, gunakan Binary Search (O(log N)).
